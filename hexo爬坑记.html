hexo更换github账户

發表於 2018-10-30 | 更新於: 2020-12-06 | 分類於 常用工具 | | 閱讀次數:
字數統計: 351 字 | 閱讀時長 ≈ 1 分钟

前提背景：

​ 因为前一段时间，因为个人mac的原因导致github账户两步验证失败，账户登录不上。故而申请了新的github账户，然后hexo绑定原有的github账户，进行hexo的迁移。

迁移步骤：

  1. 新建一个github账户

  2. 新建一个仓库

    • 仓库的命名规则username.github.io
    • github pages 设置默认分支

  3. 阿里云，gadayy等网站申请域名并解析

    • 申请域名
    • DNS解析

  4. 修改原hexo本地库

    • 修改本地hexo的_config.yml文件

    • source目录下CNAME文件的域名地址

  5. hexo clean &&generate&&d

  1. 遇到的坑：

    • 分支不匹配，建立新库有时候默认分支是gh-pages，有时候是master要和hexo的config文件节点匹配

    • node版本过高

      • hexo d的时候提示错误 我的是14 然后brew unlink node 解绑

      • brew seach node 查询可安装版本

      • brew install node@12 （我的是12.18可用 hexo是3.9.0）

      • brew link node@12

      • link的时候可能提示要更改路径 vim ~/.zshrc 添加export PATH=”/usr/local/opt/node@12/bin:$PATH” 如图：

  1. custom domain失效 ：source 目录下加一个CNAME文件将域名填写进去，然后hexo generate的时候会自动生成一个CNAME.html文件。hexo d就可以解决了。（切记文件没有后缀名md也没有）
